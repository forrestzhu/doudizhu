# Rule AI Autoplay Implementation Plan

**Goal:** Add a configurable rule-based decision policy, a stepwise game session
API, player-facing hints/autoplay in Electron, and trusted episode logs for
strategy optimization.

**Architecture:** Rust remains the source of truth for rules, state transitions,
visibility, policy decisions, and full game logs. Electron main owns game
sessions and complete records; renderer receives only the selected viewer's
projection plus public history and hint cards for that viewer.

**Tech Stack:** Rust core and harness, Electron main/preload, vanilla renderer,
Playwright E2E.

## Task 1: Rust Step API And Configurable Policy

**Files:**
- Modify: `src/engine.rs`
- Modify: `src/decision.rs`
- Modify: `src/lib.rs`

**Requirements:**
- Add a safe step API without exposing private `apply_decision`.
- Add a configurable `RuleBasedPolicy` implementing `DecisionPolicy`.
- Keep `LowestLegalPolicy` working as a compatibility wrapper.
- Add unit tests for turn ownership, finished games, invalid players, policy
  configuration, and avoiding bombs when a normal response exists.

## Task 2: Harness Session, Hint, Step, And Logs

**Files:**
- Modify: `src/harness.rs`
- Modify: `src/bin/harness.rs`

**Requirements:**
- Add trusted `EpisodeReport` with seed, complete initial hands, bottom cards,
  policies, public turn history, outcome, and hash.
- Add player-facing `GameViewReport` that contains only current viewer hand,
  public hand counts, role/relationship, current player, previous play, winner,
  bottom cards, and public history.
- Add `HintReport` generated by the same configurable policy without mutating
  state.
- Add CLI commands suitable for Electron IPC smoke checks:
  - `--session --seed <n> --viewer <id> --format json`
  - `--trace --seed <n> --format json`
- Preserve existing `--deal`, `--games`, and `--scenario` behavior.

## Task 3: Electron Session UI

**Files:**
- Modify: `electron/main.js`
- Modify: `electron/preload.js`
- Modify: `renderer/index.html`
- Modify: `renderer/app.js`
- Modify: `renderer/styles.css`

**Requirements:**
- Main process owns `gameId -> session` and calls Rust harness/session APIs.
- Preload exposes `startGame`, `setViewer`, `getHint`, `autoStep`.
- Renderer supports:
  - Start/redeal game.
  - Viewer switching without starting a new deal.
  - Hint button that highlights only current viewer's recommended cards.
  - Single step button for deterministic debugging.
  - Autoplay toggle that repeatedly steps until stopped or winner exists.
  - Public history list.
- Renderer must not cache or expose non-viewer hands.
